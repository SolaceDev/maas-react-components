<Subtitle>Component Description</Subtitle>

The SolaceAccordion component provides an expandable/collapsible container that helps organize content into sections. It's useful for reducing visual clutter by hiding content until needed, allowing users to focus on specific information.

The component implements a progressive disclosure pattern with the purpose of only revealing essential information up front, and moving secondary/complex content out of the main interface into an expandable area. This pattern keeps users on track of the main task while still making additional information available.

<Subtitle>When To Use It</Subtitle>

Use accordions when you have multiple sections of related content that don't need to be visible simultaneously. This interaction pattern is particularly effective for organizing large amounts of content without overwhelming the user interface.

Common use cases include FAQ sections, settings panels with advanced options, table row details, list item expansions, and card-based layouts where additional information needs to be accessible but not always visible.

<Subtitle>When Not To Use It</Subtitle>

- The content is critical and should always be visible to users
- There are only a few short sections that could easily fit on the screen at once
- Users need to compare information across multiple sections simultaneously
- The accordion would create an unnecessary extra step for frequently accessed content
- The content has a natural sequential flow that users should follow step by step
- The collapsed state might hide important information that users need to see immediately
