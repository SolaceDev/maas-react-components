<Subtitle>Component Description</Subtitle>

The Solace Checkbox component provides a customizable checkbox input for user selection. It's useful for allowing users to make binary choices, enabling or disabling options, and creating lists of selectable items.

The component implements a form input pattern with the purpose of capturing user preferences or acknowledgments. This pattern helps users make clear decisions while providing visual feedback and supporting various states such as checked, unchecked, and indeterminate.

<Subtitle>When To Use It</Subtitle>

Use Solace Checkbox when you need to offer users a binary choice or multiple selection options. This input pattern is particularly effective for:

- Allowing users to agree to terms and conditions
- Creating lists of options where multiple selections are allowed
- Toggling features or settings on and off
- Implementing "select all" functionality with child checkboxes
- Capturing user preferences in forms or surveys
- Creating task lists with completable items

<Subtitle>When Not To Use It</Subtitle>

- When users can only choose one option from a list (use radio buttons instead)
- For submitting forms or triggering immediate actions (use buttons instead)
- When the options are mutually exclusive (use toggle switches or radio buttons)
- For displaying read-only information (use text or icons instead)
- When there are more than 7-10 options (consider a multi-select dropdown or list)
- For complex data entry (use specialized input fields)
- When immediate feedback or action is required upon selection (use toggle switches)
- If the meaning of checked/unchecked states might be ambiguous to users

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiCheckbox-root` - Root checkbox container
- `.MuiCheckbox-checked` - Applied when the checkbox is checked
- `.MuiCheckbox-indeterminate` - Applied when the checkbox is in an indeterminate state
- `.MuiFormControlLabel-root` - Container for the checkbox and its label

**State Classes:**

- `.Mui-checked` - Applied when the checkbox is checked
- `.Mui-disabled` - Applied when the checkbox is disabled
- `.Mui-error` - Applied when the checkbox has an error state
- `.Mui-focused` - Applied when the checkbox has focus

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.primary` - Color for the checked state
- `theme.palette.ux.secondary` - Color for the unchecked state
- `theme.palette.ux.error` - Color for error states
- `theme.palette.ux.warning` - Color for warning states

**Layout Specifications:**

- Checkbox size: 18x18 pixels (default)
- Label padding: 9px left of the checkbox
- Transition duration: 150ms for state changes
