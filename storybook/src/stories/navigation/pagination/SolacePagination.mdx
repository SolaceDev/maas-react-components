<Subtitle>Component Description</Subtitle>

The Solace Pagination component provides navigation controls for large datasets split across multiple pages. It displays page numbers, navigation arrows, and result counts, allowing users to navigate through paginated content efficiently while maintaining awareness of their position within the dataset.

The component implements a pagination pattern with the purpose of breaking large datasets into manageable chunks. This pattern helps improve performance and user experience when dealing with extensive lists or tables of data.

<Subtitle>When To Use It</Subtitle>

Use Solace Pagination when you need to navigate through large datasets split across multiple pages. This pagination pattern is essential for managing extensive content.

Common use cases include:

- Data tables with many rows of information
- Search results spanning multiple pages
- Product listings or catalogs
- Content feeds or blog post listings
- API responses with paginated data
- Large lists that would be overwhelming as a single page

<Subtitle>When Not To Use It</Subtitle>

- For small datasets that fit comfortably on one page - show all items instead
- For real-time data feeds - use infinite scroll or live updates instead
- For step-by-step processes - use stepper components instead
- When users need to see all data at once for comparison - provide "show all" option
- For mobile apps where infinite scroll is more natural
- When data is hierarchical rather than sequential - use tree navigation

**Theme Customization:**

**Color Theming:**
The component uses the theme.palette.ux color system for all color-related styling. Key color theme paths:

- theme.palette.ux.primary.wMain - Active page background
- theme.palette.ux.primary.text.w10 - Active page text color
- theme.palette.ux.primary.text.wMain - Default text colors
- theme.palette.ux.secondary.w10 - Hover backgrounds
- theme.palette.ux.secondary.text.w50 - Disabled button text

To customize colors, override the palette.ux values in your theme configuration by extending the base theme and modifying the specific color group and variant you want to change.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(2) - Container spacing between pagination and content
- theme.spacing(0.5) - Item spacing between pagination items
- theme.components.MuiPagination.styleOverrides.root - Override pagination container styling
- theme.components.MuiPaginationItem.styleOverrides.root - Override pagination item styling
- theme.typography.body2 - Result count text styling

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and pagination behavior through pagination overrides.

**Layout Specifications:**

- Container spacing: theme.spacing(2) between pagination and content
- Item spacing: theme.spacing(0.5) between pagination items
- Page item size: 32px x 32px for consistent click targets
- Border radius: 4px for page items
- Result text: Typography body2 variant for result counts
- Responsive breakpoints: Simplified navigation on mobile devices
