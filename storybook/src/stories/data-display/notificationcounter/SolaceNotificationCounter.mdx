<Subtitle>Component Description</Subtitle>

The SolaceNotificationCounter component provides a visual badge that displays numeric values or short text to indicate counts, status updates, or notifications. It's useful for drawing user attention to important information like unread messages, pending tasks, or system alerts, allowing users to quickly identify areas that require their attention.

The component implements an animated notification badge pattern with the purpose of providing immediate visual feedback about quantifiable information. This pattern helps users stay informed about system state changes while offering customizable animations and styling to match different urgency levels and design requirements.

<Subtitle>Usage</Subtitle>

- **Placement:** Typically positioned at the top-right of the element they relate to (e.g., an icon, button, or navigation item).
- **Purpose:** Used to indicate a numerical count of items, such as unread notifications, new messages, or items in a cart.
- **Visibility:** Can be conditionally displayed based on the count (e.g., hidden when the count is zero).
- **Content:** Usually displays a number, but can also be a simple dot for a non-numeric indication.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(1) - Padding around counter value
- theme.typography.body2 - Base typography for counter value
- theme.components.MuiBadge.styleOverrides.badge - Custom badge styling
- theme.transitions.create(['transform', 'opacity']) - Animation transitions

Available mixins for consistent styling:

- theme.mixins.notification.pulse - Pulse animation effect
- theme.mixins.notification.fade - Opacity transition effect

**Layout Specifications:**

- Badge dimensions: `21px` × `21px` (default)
- Font size: `14px` (default), dynamically calculated as `(size / 3) * 2`
- Line height: `size + 1px`
- Animation duration: `1000ms`
- Pulse animation: Scale 1.00 → 1.05
- Initial animation: 1 cycle
- Update animation: 3 cycles
- Opacity transition: 0 ↔ 1 (controlled by showing state)
