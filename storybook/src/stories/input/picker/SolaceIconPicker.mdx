<Subtitle>Component Description</Subtitle>

The Solace Icon Picker component provides a customizable interface for selecting icons from a predefined set. It's useful for applications that need icon selection capabilities, allowing users to browse and choose icons with visual preview and search functionality.

The component implements an intuitive icon selection pattern with the purpose of making icon choice efficient and user-friendly. This pattern helps users quickly find and select appropriate icons while maintaining a clean and organized interface.

<Subtitle>When To Use It</Subtitle>

Use Solace Icon Picker when you need to provide users with a way to select icons from a predefined set. This picker pattern is particularly effective for interfaces requiring icon selection capabilities.

Common use cases include:

- Customization interfaces where users can select icons for their content
- Form builders that allow icon selection for elements
- Settings panels where icons can be chosen for various features
- Menu or navigation builders requiring icon selection
- Profile or avatar customization with icon options
- Content creation tools needing icon integration

<Subtitle>When Not To Use It</Subtitle>

- When only a single fixed icon is needed (use a static icon component)
- For image upload functionality (use a file uploader component)
- When custom icon upload is required (use an image upload component)
- In space-constrained interfaces (consider a simpler icon selection method)
- When icons aren't part of the core functionality (use simpler alternatives)
- For purely decorative purposes where user selection isn't needed

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.picker-container` - Root container for the picker
- `.picker-grid` - Grid layout for icon display
- `.picker-search` - Search input container
- `.picker-item` - Individual icon item container
- `.picker-item-selected` - Applied to the selected icon

**State Classes:**

- `.picker-item-hover` - Applied on icon item hover
- `.picker-item-disabled` - Applied to disabled icon items
- `.picker-search-active` - Applied when search is active
- `.picker-empty` - Applied when no icons match search
- `.picker-loading` - Applied during loading states
