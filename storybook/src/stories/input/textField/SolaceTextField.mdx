<Subtitle>Component Description</Subtitle>

The Solace Text Field component is a versatile and customizable single-line text input field designed for use in forms and other UI elements. It provides a consistent way to collect user input while offering various styling options, validation states, and customization features.

This component extends the functionality of a standard HTML input field, adding features such as label support, error/warning states, helper text, custom icons, and adornments. It's particularly useful for creating accessible and visually consistent form layouts that require single-line text input.

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiTextField-root` - Root text field container
- `.MuiInputBase-root` - Input base wrapper
- `.MuiInputBase-input` - Input element
- `.MuiInputLabel-root` - Input label element
- `.MuiFormHelperText-root` - Helper text element

**State Classes:**

- `.Mui-focused` - Applied when the text field has focus
- `.Mui-disabled` - Applied when the text field is disabled
- `.Mui-error` - Applied when the text field has an error state
- `.Mui-required` - Applied when the text field is required

**Variant Classes:**

- `.MuiTextField-outlined` - Outlined variant styling
- `.MuiTextField-filled` - Filled variant styling
- `.MuiTextField-standard` - Standard variant styling

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(4) - Input field height
- theme.spacing(2) - Padding inside input field
- theme.typography.body1 - Text style for input value
- theme.typography.body2 - Text style for label and helper text
- theme.breakpoints.values.sm - Responsive behavior breakpoint
- theme.components.MuiTextField.styleOverrides.root - Root container overrides
- theme.components.MuiInputBase.styleOverrides.input - Input element overrides

Available mixins for consistent styling:

- theme.mixins.textField.outlined - Standard outlined text field styling patterns
- theme.mixins.textField.filled - Standard filled text field styling patterns

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior using breakpoint values.

**Layout Specifications:**

- Input field height: `theme.spacing(4)` (32px)
- Label font size: `14px` (shrunk to `12px` when focused/filled)
- Helper text font size: `12px`
- Border radius: `4px`
- Transition timeout: `200ms`
