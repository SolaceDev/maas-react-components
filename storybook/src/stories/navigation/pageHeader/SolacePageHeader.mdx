<Subtitle>Component Description</Subtitle>

The Solace Page Header component provides a comprehensive header layout for application pages, combining title, subtitle, breadcrumbs, tabs, actions, and environment information. It serves as the primary navigation and context element at the top of pages, helping users understand their current location and available actions.

The component implements a page header pattern with the purpose of establishing page context and providing primary navigation elements. This pattern helps users orient themselves within the application and access key functionality related to the current page.

<Subtitle>When To Use It</Subtitle>

Use Solace Page Header as the primary header component for application pages. This header pattern is essential for establishing page context and navigation.

Common use cases include:

- Main application pages requiring title and navigation context
- Detail pages with breadcrumb navigation back to parent sections
- Pages with multiple tabs or view modes
- Pages requiring action buttons or menus in the header
- Multi-environment applications showing current environment context
- Pages with hierarchical navigation structures

<Subtitle>When Not To Use It</Subtitle>

- For modal dialogs or popups - use modal headers instead
- For simple forms without complex navigation - use simpler header components
- For landing pages or marketing content - use custom headers
- When space is extremely constrained - consider simplified headers
- For print layouts - headers may not be appropriate
- In embedded widgets or components within larger pages

**Theme Customization:**

**Color Theming:**
The component uses the theme.palette.ux color system for all color-related styling. Key color theme paths:

- theme.palette.ux.primary.text.wMain - Primary text colors for title and subtitle
- theme.palette.ux.background.w10 - Background colors for header container
- theme.palette.ux.secondary.w40 - Border and divider colors
- theme.palette.ux.brand.wMain - Environment chip and border-top color coordination

To customize colors, override the palette.ux values in your theme configuration by extending the base theme and modifying the specific color group and variant you want to change.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(2, 3) - Header padding for consistent margins
- theme.spacing(1) - Row spacing between header rows and action buttons
- theme.typography.h1 - Main title typography styling
- theme.typography.subtitle1 - Subtitle typography styling
- theme.breakpoints.down('md') - Responsive behavior breakpoints

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior through breakpoint utilities.

**Layout Specifications:**

- Header padding: theme.spacing(2, 3) for consistent margins
- Row spacing: theme.spacing(1) between header rows
- Border-top: 2px solid with environment-matched color
- Action spacing: theme.spacing(1) between action buttons
- Responsive breakpoints: Proper stacking on mobile devices
- Title typography: H1 for main title, subtitle styling for secondary text
