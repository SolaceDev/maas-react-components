<Subtitle>Component Description</Subtitle>

The Solace Radio component provides a single radio button option for use within radio groups or as standalone selection controls. It's useful for allowing users to select one option from a set of mutually exclusive choices, with support for labels, sub-text, and various visual states.

The component implements the standard radio button interaction pattern with the purpose of enabling single-selection from grouped options. This pattern helps users understand that only one choice can be made while providing clear visual feedback for the selected state.

<Subtitle>When To Use It</Subtitle>

Use Solace Radio when you need individual radio button options within a radio group or for standalone binary choices. This selection pattern is particularly effective for forms where users must choose exactly one option from a set.

Common use cases include:

- Form fields requiring single selection from multiple options (payment methods, shipping options)
- Settings pages where users choose one configuration option (theme selection, notification preferences)
- Survey questions with mutually exclusive answers (yes/no, rating scales)
- Filtering interfaces where only one filter can be applied at a time
- Configuration wizards where users select one path or option to proceed

<Subtitle>When Not To Use It</Subtitle>

- For multiple selections - use checkboxes instead as they allow multiple choices
- When you have only two options - consider using a toggle switch for better UX
- For actions that trigger immediate effects - use action buttons instead
- When you have more than 7-8 options - use a dropdown select for better space efficiency
- For navigation between sections - use tabs or navigation links instead
- When the options are not mutually exclusive - use checkboxes to allow multiple selections

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiRadio-root` - Root radio button container
- `.MuiInputLabel-root` - Label element styling
- `.MuiFormLabel-root` - Sub-text styling

**State Classes:**

- `.Mui-checked` - Applied when the radio button is selected
- `.Mui-disabled` - Applied when the radio button is disabled
- `.Mui-focusVisible` - Applied when the radio button receives keyboard focus
- `.readOnly` - Applied when the radio button is in read-only state

**Custom Classes:**

- `.light-sub-text` - Applied when `lightSubText` prop is true for lighter sub-text color
- `.radio-btn-label` - Applied to sub-text elements for consistent styling

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.primary.text.wMain` - Label text color
- `theme.palette.ux.secondary.text.w50` - Disabled text color
- `theme.typography.subtitle1` - Large label typography
- `theme.typography.body1` - Default label typography

**Layout Specifications:**

- Grid layout: `auto 1fr` columns for radio and content
- Grid rows: `auto auto` for label and sub-text
- Radio button height: `24px` (fixed line height for alignment)
- Label font weight: `medium` for large labels or when sub-text is present, `regular` otherwise
- Sub-text positioning: Grid column 2, row 2 for proper alignment
