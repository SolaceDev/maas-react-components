<Subtitle>Component Description</Subtitle>

The SolaceGridListMultiSelect component extends the GridList functionality by adding multi-selection capabilities through checkboxes. It's useful for scenarios where users need to select multiple items from a structured grid layout, allowing for efficient batch operations and data management.

The component implements a selectable grid pattern with the purpose of enabling bulk actions on tabular data. This pattern helps users manage multiple selections while maintaining the organized structure of a grid layout, making it ideal for administrative interfaces and data management tools.

<Subtitle>When To Use It</Subtitle>

Use SolaceGridListMultiSelect when you need to provide users with the ability to select multiple items from a grid-based layout. This selection pattern is particularly effective for batch operations and data management.

Common use cases include:

- Bulk operations on event meshes, environments, or other business entities
- Multi-item selection for batch processing or updates
- Administrative interfaces requiring multiple item management
- Data management tools with bulk action capabilities
- Resource selection interfaces for batch operations
- List management with checkbox-based selection

<Subtitle>When Not To Use It</Subtitle>

- For simple data display without selection needs - use SolaceGridList instead
- When single-item selection is sufficient - use standard GridList with clickable rows
- For read-only data presentation - use basic grid layouts without selection
- When complex table features are needed - use SolaceTable for full table functionality
- For simple unstructured lists - use SolaceList for basic item listings
- When selection is not a primary user action - use simpler display components

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(2) - Default padding between grid items
- theme.typography.body1 - Base typography for grid content
- theme.components.MuiCheckbox.styleOverrides.root - Custom checkbox styling
- theme.components.MuiImageList.styleOverrides.root - Custom grid container styling
- theme.transitions.create(['background-color']) - Selection state transitions

Available mixins for consistent styling:

- theme.mixins.gridList.row - Standard row styling
- theme.mixins.gridList.selected - Selected state styling
- theme.mixins.gridList.checkbox - Checkbox container styling

**Layout Specifications:**

- Grid template columns: Defined by the `gridTemplate` prop using CSS Grid syntax
- Item height: Calculated dynamically using `getGridListItemHeight()` utility
- Virtual list height: Configurable through `virtualizedListOption.height`
- Checkbox column width: Fixed at `48px`
- Row padding: `8px` (theme.spacing(1))
- Selection transition: `200ms` background-color fade
- Hover state opacity: 0.04
- Selected state opacity: 0.08
- Checkbox position: Left-aligned with `16px` margin
