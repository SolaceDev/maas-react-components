<Subtitle>Component Description</Subtitle>

The SolaceNotificationCounter component provides a visual badge that displays numeric values or short text to indicate counts, status updates, or notifications. It's useful for drawing user attention to important information like unread messages, pending tasks, or system alerts, allowing users to quickly identify areas that require their attention.

The component implements an animated notification badge pattern with the purpose of providing immediate visual feedback about quantifiable information. This pattern helps users stay informed about system state changes while offering customizable animations and styling to match different urgency levels and design requirements.

<Subtitle>When To Use It</Subtitle>

Use SolaceNotificationCounter when you need to display numeric values or status indicators that represent countable items or states requiring user attention. This interaction pattern is particularly effective for alerting users to new or updated content without being overly intrusive.

Common use cases include:

- Unread message counts in messaging applications or notification systems
- Pending task counters in workflow management or todo applications
- New item indicators for feeds, dashboards, or content management systems
- Error or warning counts in system monitoring or validation interfaces
- Shopping cart item counts in e-commerce applications
- Status badges showing the number of active connections, processes, or system states

<Subtitle>When Not To Use It</Subtitle>

- Static information that doesn't change or require user attention (use regular text or labels instead)
- Complex data that requires more context than a simple number or short text can provide
- Primary navigation elements where the badge might distract from the main interface flow
- Large numbers that won't fit comfortably in a circular badge format (consider alternative layouts)
- Time-sensitive information that requires immediate action (use alerts or modal dialogs for critical notifications)
- Decorative purposes where the badge doesn't represent actual actionable data
- Mobile interfaces where small badges might be difficult to read or interact with
- Accessibility-critical scenarios where color or animation might not be perceivable by all users

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.notificationCounter` - Root badge container with custom Solace styling
- `.MuiTooltip-root` - Tooltip wrapper (when title prop is provided)

**Custom Classes:**

- `.Badge` - Styled badge container with size and font size customization
- `.Value` - Text content wrapper with opacity transitions

**Theme Customization:**

The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.mixins.component_NotificationCounter.container` - Badge container styling
- `theme.mixins.component_NotificationCounter.value` - Text value styling
- `theme.palette.ux.info.w100` - Pulse animation colors

**Layout Specifications:**

- Default size: `21px` (width and height)
- Default font size: `14px`
- Default animation duration: `1000ms`
- Line height: Calculated as `size + 1` pixels
- Font size calculation: `(size / 3) * 2` when not explicitly set
- Pulse animation: Scale from 1 to 1.05 with box-shadow effect
- Initial animation repeats: `1` cycle
- Update animation repeats: `3` cycles
- Show/hide opacity transition: Controlled by `showing` state
