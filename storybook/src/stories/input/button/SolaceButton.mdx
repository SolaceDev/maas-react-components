<Subtitle>Component Description</Subtitle>

The Solace Button component is a versatile and customizable button for use in all Solace-based applications. It provides various visual styles, sizes, and states to accommodate different user interface needs. The button can be used for actions, navigation, form submission, and more.

The component implements a standard button pattern with the purpose of triggering actions or events within the application. This pattern helps users interact with the interface, submit forms, navigate between pages, and perform various operations.

<Subtitle>When To Use It</Subtitle>

Use Solace Button when you need to provide users with a clear, interactive element for performing actions. This button component is particularly effective for:

- Submitting forms or data
- Triggering modal dialogs or popups
- Navigating between pages or sections
- Executing actions like save, delete, or update
- Toggling features or settings
- Initiating processes or workflows
- Creating call-to-action elements
- Providing clear, clickable targets for user interaction

<Subtitle>When Not To Use It</Subtitle>

- For long blocks of text or complex content (use a card or panel instead)
- When a less prominent action is needed (consider using a link)
- For grouping multiple related actions (use a dropdown or menu)
- When toggle functionality is required (use a switch or checkbox)
- For selecting from a list of options (use radio buttons or a select component)
- When the action requires a drag-and-drop interaction
- For displaying static, non-interactive information

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiButton-root` - Root button container
- `.MuiButton-text` - Applied to text variant buttons
- `.MuiButton-outlined` - Applied to outline variant buttons
- `.MuiButton-contained` - Applied to call-to-action variant buttons
- `.MuiButton-startIcon` - Container for the start icon
- `.MuiButton-endIcon` - Container for the end icon

**State Classes:**

- `.Mui-disabled` - Applied when the button is disabled
- `.Mui-focusVisible` - Applied when the button has keyboard focus
- `.MuiButton-loading` - Applied when the button is in a loading state

**Size Classes:**

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(1) - Padding for small buttons
- theme.spacing(2) - Padding for medium buttons
- theme.spacing(3) - Padding for large buttons
- theme.typography.button - Text style for button labels
- theme.breakpoints.values.sm - Responsive behavior breakpoint
- theme.components.MuiButton.styleOverrides.root - Root container overrides
- theme.components.MuiButton.styleOverrides.contained - Contained variant overrides

Available mixins for consistent styling:

- theme.mixins.button.primary - Standard primary button styling patterns
- theme.mixins.button.secondary - Standard secondary button styling patterns

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior using breakpoint values.

**Layout Specifications:**

- Small button padding: `6px 16px`
- Medium button padding: `8px 24px`
- Large button padding: `12px 32px`
- Icon spacing: `8px` between icon and text
- Border radius: `4px`
- Transition duration: `250ms`
