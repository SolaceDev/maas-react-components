<Subtitle>Component Description</Subtitle>

The SolaceDrawer component provides a sliding panel that enters from the edge of the screen to display additional content, navigation options, or contextual information without navigating away from the current view. It's useful for organizing supplementary content that supports the main workflow, allowing users to access additional tools and information when needed.

The component implements a persistent drawer pattern with the purpose of creating a non-modal overlay that can coexist with the main content. This pattern provides additional workspace without completely interrupting the user's current task while offering optional resizing capabilities for user customization.

<Subtitle>Usage</Subtitle>

- Drawers provide additional functionality or information related to the main page content without navigating away or blocking interaction with the page.
- Use drawers for tasks that are too complex or would take up too much space if embedded directly on the page (e.g., detailed settings, selection from a large list with filtering).
- Drawers are typically triggered by a user action, appear locked to one edge of the screen, and slide in over the page content.
- Consider that drawer content is often temporary; if frequent switching or persistent input is needed, alternative patterns might be more suitable.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing([NUMBER]) - Padding and margin (e.g., theme.spacing(1) for 8px content margin)
- theme.typography.[VARIANT] - Text styles for drawer content
- theme.breakpoints.values.[SIZE] - Responsive behavior breakpoints
- theme.components.MuiDrawer.styleOverrides.root - Custom styling for drawer container
- theme.components.MuiDrawer.styleOverrides.paper - Custom styling for drawer paper/content area

Available mixins for consistent styling:

- theme.mixins.toolbar - Consistent toolbar styling if applicable in drawer content

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior using breakpoint values.

**Layout Specifications:**

- Default width: `320px`
- Minimum width: `100px` (when resizable)
- Maximum width: `1000px` (when resizable)
- Resize handle width: `8px`
- Border radius: `2px`
- Content margin: `theme.spacing(1)` (8px, left or right based on anchor)
- Box shadow: `0px 2px 4px` with theme color
- Variant: `persistent` (non-modal)
- Transition timeout: `225ms`
