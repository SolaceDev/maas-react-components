<Subtitle>Component Description</Subtitle>

The Solace Categorized Search component provides a powerful search interface that combines text-based searching with category filtering. It's useful for narrowing down search results within specific categories, allowing users to efficiently find content across different sections or types of data.

The component implements a combined search pattern with the purpose of enhancing search precision through categorization. This pattern helps users quickly locate relevant information by combining free-text search with predefined category filters, while maintaining a clean and intuitive interface.

<Subtitle>When To Use It</Subtitle>

Use Solace Categorized Search when you need to provide users with a refined search experience. This search pattern is particularly effective for:

- Searching through large datasets with distinct categories
- Filtering content across multiple sections or types
- Implementing advanced search functionality in dashboards
- Creating searchable documentation or knowledge bases
- Managing multi-faceted content repositories
- Building admin interfaces with categorized data
- Implementing e-commerce product search

<Subtitle>When Not To Use It</Subtitle>

- When the dataset is small or simple enough for basic text search
- If there are no meaningful categories to filter by
- When users primarily need exact matches (use a dropdown or select instead)
- For simple yes/no filtering (use checkboxes or toggle switches)
- When search categories would add unnecessary complexity
- If the search scope is limited to a single category
- When real-time search results are required (consider a simpler search input)
- If the primary interaction is data entry rather than search

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiTextField-root` - Root search input container
- `.MuiToggleButtonGroup-root` - Category buttons container
- `.MuiToggleButton-root` - Individual category button
- `.MuiFormHelperText-root` - Helper text container

**State Classes:**

- `.Mui-focused` - Applied when the search input has focus
- `.Mui-selected` - Applied to the currently selected category button
- `.Mui-disabled` - Applied when the component is disabled
- `.Mui-error` - Applied when the component has an error state

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.primary` - Primary color for active elements
- `theme.palette.ux.secondary` - Secondary color for inactive elements
- `theme.palette.ux.error` - Color for error states
- `theme.palette.ux.warning` - Color for warning states

**Layout Specifications:**

- Search input padding: Material-UI's default text field padding
- Category button spacing: 4px between buttons
- Button height: Matches the height of the search input
- Transition duration: 200ms for hover and focus effects
