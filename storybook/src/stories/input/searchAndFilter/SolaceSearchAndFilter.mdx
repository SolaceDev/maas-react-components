<Subtitle>Component Description</Subtitle>

The Solace Search and Filter component is a specialized text input field designed for search and filtering functionality. It extends the SolaceTextField component with additional features specific to search and filter operations, including appropriate icons and a clear button.

This component provides a consistent way to implement search and filter interfaces across Solace applications. It includes built-in support for both search and filter modes, each with their own distinctive icon, and maintains focus after clearing the input for a smoother user experience.

<Subtitle>When To Use It</Subtitle>

Use Solace Search and Filter in the following scenarios:

- Implementing search functionality in applications
- Creating filter inputs for data tables or lists
- When you need a text input with a clear button
- For search interfaces that should maintain focus after clearing
- When you need visual distinction between search and filter modes
- In data-heavy interfaces where users need to find specific content
- When you want to provide immediate visual feedback for search/filter actions
- For implementing consistent search patterns across your application

<Subtitle>When Not To Use It</Subtitle>

- For regular text input fields (use SolaceTextField instead)
- When advanced search capabilities are needed (use SolaceCategorizedSearch instead)
- For multi-select filtering (use SolaceSelect or SolaceCheckBox)
- When you need a complex search form with multiple fields
- If you need to preserve search history (consider a more specialized component)
- When the search/filter operation requires additional UI elements
- If you need to show search suggestions or autocomplete

**Layout Specifications:**
