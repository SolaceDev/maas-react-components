<Subtitle>Component Description</Subtitle>

The Solace Side Panel Layout component provides a flexible container that manages a main content area alongside an optional collapsible side panel. The side panel can appear from either the left or right edge and can be configured to either overlay the main content or push it aside. This component is essential for creating responsive layouts that need to accommodate secondary content or navigation without losing focus on the primary interface.

The component implements a drawer pattern with the purpose of providing contextual information, navigation, or additional tools while maintaining the main workflow. It supports both persistent and temporary panel modes, making it adaptable to various use cases and screen sizes.

<Subtitle>When To Use It</Subtitle>

Use Solace Side Panel Layout when you need to display supplementary content alongside your main interface without navigating away from the current context. This layout pattern is particularly effective for responsive applications that need flexible content organization.

Common use cases include:

- Application navigation where the side panel contains menu items, user profile, or app-level settings
- Detail views where the main content shows a list and the side panel displays detailed information about selected items
- Configuration interfaces where the main area shows content and the side panel contains settings or controls
- Dashboard layouts where the side panel houses filters, search, or secondary data while the main area displays primary metrics
- Documentation or help systems where the side panel contains table of contents or related topics
- Multi-step workflows where the side panel shows progress, navigation, or contextual assistance
- Content editing interfaces where the side panel provides tools, properties, or preview functionality

<Subtitle>When Not To Use It</Subtitle>

- For simple modal dialogs or overlays - use SolacePopover or Material-UI Dialog components instead
- When the secondary content is brief and doesn't warrant a dedicated panel - consider using tooltips or inline expansion
- For mobile-first applications where screen real estate is limited - consider full-screen navigation or bottom sheet patterns
- When the side content is the primary focus - restructure to make it the main content area instead
- For temporary notifications or alerts - use SolaceToasts or notification components
- When users need to compare content side-by-side equally - use SolaceSplitPane for balanced layouts
- For complex multi-panel layouts - consider custom grid layouts or specialized dashboard components

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiDrawer-root` - Root container for the side panel drawer component
- `.MuiDrawer-paper` - The actual panel content container with styling and positioning
- `.MuiBackdrop-root` - Background overlay (hidden when overlayContent is false)

**Position Classes:**

- `.MuiDrawer-paperAnchorLeft` - Applied when side panel is positioned on the left
- `.MuiDrawer-paperAnchorRight` - Applied when side panel is positioned on the right

**State Classes:**

- `.MuiDrawer-modal` - Applied when the drawer is in temporary/overlay mode
- `.Mui-expanded` - Applied when the side panel is open and visible

**Custom Classes:**

- `#drawer-container` - Container element that manages the layout positioning and transitions
- `.main` - Main content area that responds to panel state changes

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.components.MuiDrawer.styleOverrides.root - Customize the root drawer container
- theme.components.MuiDrawer.styleOverrides.paper - Customize the drawer paper (panel content)
- theme.components.MuiBackdrop.styleOverrides.root - Customize the backdrop overlay

To customize layout and structure, override the component styleOverrides in your theme configuration.

**Layout Specifications:**

- Container display: flex with full width and height (100%)
- Container position: relative with overflow: hidden for proper clipping
- Panel width transition: 0.25s for smooth width changes when toggling
- Panel positioning: absolute within the container for proper layering
- Main content margin: Dynamic based on panel state, position, and overlay mode
- Flex direction: row-reverse when panel is positioned on the right side
- Panel paper width: Consistent with sidePanelWidth prop for proper sizing
- Backdrop display: none when using persistent (non-overlay) mode

**Transition Specifications:**

- Panel transition duration (opening): theme.transitions.duration.enteringScreen (225ms)
- Panel transition duration (closing): theme.transitions.duration.leavingScreen (195ms)
- Panel transition easing: theme.transitions.easing.easeOut for smooth animations
- Default panel width: 320px when no width is specified

**Accessibility Considerations:**

- Proper focus management when opening/closing the panel
- Keyboard navigation support for panel content
- ARIA attributes for describing the panel's state (expanded/collapsed)
- Screen reader announcements for panel open/close actions
