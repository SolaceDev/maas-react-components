<Subtitle>Component Description</Subtitle>

The Solace Colors component provides a comprehensive color palette system based on the Solace design system with semantic color tokens for consistent theming. It's useful for maintaining brand consistency and accessibility across applications, implementing a systematic color approach that supports multiple themes, semantic meanings, and accessibility requirements while providing clear guidelines for color usage.

<Subtitle>When To Use It</Subtitle>

Use Solace Color Palette when you need to reference or showcase the complete color scheme of the Solace design system. This component is particularly effective for design and development contexts.

Common use cases include:

- During the design phase to ensure consistency across UI elements
- When developing new components and need to reference the correct color values
- For documentation purposes to provide a comprehensive view of the color system
- As a quick reference tool for designers and developers working with the Solace theme

<Subtitle>When Not To Use It</Subtitle>

- When you only need to display a single color or a small subset of colors
- In production applications where end-users don't need access to the entire color palette
- When you need to allow users to modify or create custom color palettes
- If you're working with a different theme system that doesn't align with the Solace color structure
- In performance-critical parts of an application where rendering the full palette might impact load times
- When you need to display color information in a different format (e.g., as a color picker or in a table format)

**Theme Customization:**

**Color Theming:**
The component uses the theme.palette.ux color system for all color-related styling. Key color theme paths:

- theme.palette.ux.[COLOR_GROUP].[COLOR_VARIANT] - Used for displaying and copying color values
- theme.palette.ux.primary.text.wMain - Primary text color for labels
- theme.palette.ux.secondary.text.wMain - Secondary text color for descriptions
- theme.palette.ux.background.wMain - Background color for sections
- theme.palette.ux.secondary.w10 - Hover background for interactive elements

To customize colors, override the palette.ux values in your theme configuration by extending the base theme and modifying the specific color group and variant you want to change.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(2) - Main container padding (16px)
- theme.spacing(1) - Color item margin bottom (8px)
- theme.spacing(2.5) - Section margin bottom (20px)
- theme.shape.borderRadius - Border radius for sections (8px)
- theme.components.MuiBox.styleOverrides.root - Override container styling
- theme.typography.body2 - Typography styling for color labels

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and section appearance through box overrides.

**Layout Specifications:**

- Color swatch dimensions: 32px x 32px
- Main container padding: theme.spacing(2) (16px)
- Color item margin bottom: theme.spacing(1) (8px)
- Section margin bottom: theme.spacing(2.5) (20px)
- Border radius for sections: theme.shape.borderRadius (8px)
