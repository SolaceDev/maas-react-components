<Subtitle>Component Description</Subtitle>

The Solace Breadcrumb component provides navigation breadcrumb links that help users understand their current location within an application hierarchy. It's useful for displaying navigation paths as clickable buttons, allowing users to navigate back to parent routes easily.

The component implements hierarchical navigation with the purpose of providing clear wayfinding. This pattern helps users understand where they are in the application structure while still maintaining the ability to quickly navigate to any parent level.

<Subtitle>When To Use It</Subtitle>

Use Solace Breadcrumb when you have multi-level navigation hierarchies. This navigation pattern is particularly effective for applications with deep page structures.

Common use cases include:

- Multi-step forms or wizards where users need to navigate between steps
- File system or folder-like navigation structures
- E-commerce category navigation (Home > Category > Subcategory > Product)
- Administrative interfaces with nested sections and subsections

<Subtitle>When Not To Use It</Subtitle>

- For single-level navigation - use a standard navigation menu instead
- When the navigation hierarchy is shallow (only 1-2 levels) - consider using tabs or simple back buttons
- For mobile applications with limited screen space - breadcrumbs can be too small to interact with effectively
- When users typically don't need to navigate backwards in the flow - such as checkout processes
- For applications where the navigation path is not linear or hierarchical
- When screen real estate is at a premium and the navigation path is not critical to user understanding

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiBreadcrumbs-root` - Root breadcrumb container
- `.MuiBreadcrumbs-li` - Individual breadcrumb list item
- `.MuiBreadcrumbs-separator` - Separator between breadcrumb items

**State Classes:**

- `.Mui-disabled` - Applied when breadcrumb links are disabled
- `.MuiLink-underlineHover` - Applied to links that show underline on hover
- `.MuiLink-underlineNone` - Applied to current page link (no underline)

**Custom Classes:**

- Custom styled `SolaceLink` with max-width: 200px and text overflow ellipsis
- Custom `RouteContainer` with flex layout and column gap

**Theme Customization:**

**Color Theming:**
The component uses the theme.palette.ux color system for all color-related styling. Key color theme paths:

- theme.palette.ux.primary.wMain - Active breadcrumb link color
- theme.palette.ux.secondary.text.wMain - Current page text color (no link)
- theme.palette.ux.primary.w90 - Link hover color
- theme.palette.ux.secondary.wMain - Separator icon color

To customize colors, override the palette.ux values in your theme configuration by extending the base theme and modifying the specific color group and variant you want to change.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(0.5) - Route container gap between breadcrumb elements
- theme.components.MuiBreadcrumbs.styleOverrides.root - Override breadcrumb container styling
- theme.components.MuiLink.styleOverrides.root - Override link styling
- theme.typography.body2 - Breadcrumb text styling

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and link behavior through link overrides.

**Layout Specifications:**

- Link max-width: 200px
- Text overflow: ellipsis with whiteSpace: nowrap
- Route container: flex with row direction and center alignment
- Default maxItems: 8
- NavigateNext icon as separator with fontSize="small"
