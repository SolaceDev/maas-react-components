`SolacePicker`

<Subtitle>Component Description</Subtitle>

The Solace Color Picker component provides a grid-based selection interface for choosing colors from predefined color palettes. It's useful for applications where users need to select visual identifiers like environment colors, status indicators, or category colors, allowing users to make selections from an organized grid layout with visual color previews.

The component implements a custom dropdown select pattern with the purpose of making color selection intuitive through visual grid layouts. This pattern helps users quickly identify and select colors through visual representation while maintaining form consistency and providing accessible color selection workflows.

<Subtitle>When To Use It</Subtitle>

Use Solace Color Picker when you need visual color selection from predefined color sets in a grid format. This selection pattern is particularly effective for applications requiring visual categorization and color-based identification.

Common use cases include:

- Environment selection where colors represent different deployment stages (development, staging, production)
- Status indicator selection for tagging items with visual state representations using color coding
- Category assignment using color-coded classification systems for organizing content
- Theme customization interfaces where users select from predefined brand color palettes
- Visual workflow configuration where colors represent different process stages or states
- Badge and label color assignment for content categorization and visual organization

<Subtitle>When Not To Use It</Subtitle>

- For custom color selection with full RGB/HSL controls - use a comprehensive color picker component with color wheel and input fields instead
- When only text-based options are needed without visual representation - use regular SolaceSelect for non-visual selections
- For large sets of color options (>20 colors) - consider autocomplete or search-based color selection instead
- When colors don't have semantic meaning or purpose - use standard dropdown components for arbitrary non-visual choices
- For single true/false color scenarios - use radio buttons or toggle switches for binary color selections
- When accessibility requires text-only alternatives - ensure color selections have proper text equivalents and ARIA labels
- For icon selection needs - use SolacePicker with variant="icons" for icon-based selection instead

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiSelect-root` - Root select container for the color picker dropdown
- `.SolacePickerGrid` - Custom grid layout class applied to the MenuList for organizing color options
- `.MuiMenuItem-root` - Individual color option styling within the dropdown menu
- `.MuiOutlinedInput-root` - Input field container styling with outline border

**State Classes:**

- `.Mui-disabled` - Applied when color picker is disabled, graying out the component
- `.Mui-error` - Applied when picker has validation errors, showing red error styling
- `.Mui-focused` - Applied when picker receives focus, highlighting the input border
- `.Mui-selected` - Applied to currently selected color menu item

**Custom Classes:**

- `SolacePickerIconWrapper` - Icon/color container with centered alignment and fixed dimensions
- `Color` - Colored square display component with border, typography, and "Aa" preview text
- `EmptyMessageContainer` - Centered empty state message styling for when no colors are available

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.primary.text.w100` - Text color for light background color options
- `theme.palette.ux.primary.text.w10` - Text color for dark background color options
- `theme.palette.ux.background.w10` - Default light background color option
- `theme.palette.ux.accent.n2.w20` - Light blue color option
- `theme.palette.ux.accent.n1.w20` - Light accent color option
- `theme.palette.ux.accent.n6.w30` - Medium accent color option
- `theme.palette.ux.accent.n5.w60` - Darker accent color option
- `theme.palette.ux.accent.n7.wMain` - Main accent color option
- `theme.palette.ux.accent.n0.wMain` - Primary brand color option
- `theme.palette.ux.accent.n3.wMain` - Secondary accent color option
- `theme.palette.ux.accent.n4.wMain` - Tertiary accent color option
- `theme.palette.ux.accent.n9.wMain` - Dark accent color option

**Layout Specifications:**

- Color item size: `24px` (theme.spacing(3)) width and height
- Item gap: `8px` spacing between grid color items
- Default colors per row: `5` color items in grid layout
- Default rows displayed: `5` rows maximum before scrolling
- Content padding: `16px` (theme.spacing(2)) around dropdown content
- Select display padding: `4px 40px 4px 4px` for dropdown indicator spacing
- Border radius: `4px` (theme.spacing(0.5)) for color preview squares
- Typography: `caption` font size and weight for "Aa" preview text
- Menu max height: `calc(100% - 96px)` with custom height calculations
- Grid template columns: `repeat(5, 1fr)` for equal column distribution
