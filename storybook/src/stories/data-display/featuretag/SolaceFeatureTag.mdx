<Subtitle>Component Description</Subtitle>

The SolaceFeatureTag component provides a visual indicator for displaying feature states, release stages, or development status with customizable styling and colors. It's useful for showing feature availability, beta status, or experimental indicators, allowing users to quickly identify and understand the current state or maturity level of features and functionality.

The component implements a feature status pattern with the purpose of providing clear visual communication about feature readiness and availability. This pattern helps users understand what to expect from different features while maintaining consistent visual styling across different feature states and development stages.

<Subtitle>Usage Guidelines</Subtitle>

**Visual Design & Accessibility Guidelines:**

- **Contrast**: Ensure sufficient contrast between the tag's text/border and its background, especially when customizing colors.
- **Sizing**: Choose appropriate size based on context and surrounding elements
- **Placement**: Position consistently within feature lists, navigation, or content areas

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing([NUMBER]) - Padding and border radius (e.g., theme.spacing(0.5) for border radius)
- theme.typography.[VARIANT] - Text styles for tag content
- theme.breakpoints.values.[SIZE] - Responsive behavior breakpoints
- theme.components.MuiChip.styleOverrides.root - Custom styling for tag container
- theme.components.MuiChip.styleOverrides.label - Custom styling for tag text

Available mixins for consistent styling:

- theme.mixins.toolbar - Consistent toolbar styling if applicable in tag groups

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior using breakpoint values.
