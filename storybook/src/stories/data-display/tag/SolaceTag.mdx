`SolaceTag`

<Subtitle>Component Description</Subtitle>

The SolaceTag component provides a visual label element for categorizing, organizing, or highlighting content with customizable styling options. It's useful for content classification, metadata display, or visual organization systems, allowing users to quickly identify and group related information through consistent visual markers.

The component implements a flexible tagging pattern with the purpose of providing visual categorization without complex interactions. This pattern helps organize information hierarchically while offering extensive customization options for colors, variants, and visual styling to match different content types and organizational schemes.

<Subtitle>When To Use It</Subtitle>

Use SolaceTag when you need to display categorization or classification information that helps organize content visually. This interaction pattern is particularly effective for content organization and information architecture.

Common use cases include:

- Content categorization with topic tags for articles, documentation, or resources to help users filter and browse
- Metadata visualization showing content properties like difficulty levels, content types, or subject areas
- Visual organization systems for portfolios, galleries, or content libraries with thematic grouping
- Status or classification labels that don't require interaction but provide important contextual information
- Keyword highlighting in search results or content discovery interfaces
- Topic indicators for forums, discussions, or community content to facilitate browsing and discovery

<Subtitle>When Not To Use It</Subtitle>

- Interactive elements that require user actions like selection or deletion (use SolaceChip for interactive functionality)
- Navigation elements where users expect standard navigation patterns and behaviors
- Critical status information that requires immediate attention (use alerts or status indicators for urgent information)
- Temporary or frequently changing information where static tags might become outdated quickly
- Complex data relationships that require more context than a simple visual label can provide
- Accessibility scenarios where color-only differentiation might not be sufficient for all users
- Environments where visual consistency with interactive elements might confuse users about functionality
- Mobile interfaces where small tags might be difficult to read or distinguish from actionable elements

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiChip-root` - Root chip container with Material-UI styling
- `.solaceTag` - Custom class for Solace-specific tag styling
- `.MuiChip-label` - Text content wrapper
- `.MuiChip-icon` - Icon container for leading icons

**Variant Classes:**

- `.MuiChip-filled` - Applied when variant is "filled"
- `.MuiChip-outlined` - Applied when variant is "outlined"

**State Classes:**

- `.MuiChip-clickable` - Applied when clickable is true

**Custom Icon Classes:**

- `.leadingIcon` - Container for custom leading icons

**Theme Customization:**

The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.deprecated.chip` - Color mappings for borders, fills, and text colors

**Layout Specifications:**

- Height: Fixed at `sm` size (18px from CHIP_PX_HEIGHTS)
- Border radius: Fixed at `xs` size (20px from CHIP_PX_BORDER_RADIUS)
- Font size: Fixed at `xs` size (12px from BASE_FONT_PX_SIZES)
- Font weight: `400` (regular)
- Default variant: `FILLED`
- Default max width: `100%`
- Padding: `2px` left and right
- Icon size: `14x14` pixels for leading icons
- Size: Always `medium` (not responsive to content)
