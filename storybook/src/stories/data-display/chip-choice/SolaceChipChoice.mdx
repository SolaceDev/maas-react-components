<Subtitle>Component Description</Subtitle>

The Solace Chip Choice component provides a compact, interactive element for representing selectable options in a space-efficient format. It's useful for single or multiple selection scenarios, allowing users to quickly choose from a set of predefined options.

The component implements a selection-based interaction pattern with the purpose of simplifying choice interfaces. This pattern helps users clearly identify available options and their current selection state while maintaining a clean, uncluttered interface.

<Subtitle>Usage</Subtitle>

Chip choices are compact, selectable elements used for representing options in a space-efficient format. They function similar to radio buttons or checkboxes but with a more prominent, touch-friendly interface.

### When to use

- **Filter selection**: For toggling filters in search or data display interfaces
- **Option selection**: As an alternative to radio buttons for a more visual selection experience
- **Category selection**: For choosing from a set of predefined categories
- **Preference toggles**: For enabling/disabling features or settings in a compact space

### Guidelines

- Limit to 5-7 options for optimal usability
- Use consistent sizing and spacing between chip choices
- Clearly differentiate selected and unselected states
- Group related options together
- Consider using icons to enhance visual recognition
- Use the outlined variant for choice chips (Material Design recommendation)

### Accessibility

- Ensure sufficient color contrast between selected and unselected states
- Provide keyboard navigation support between options
- Use appropriate ARIA attributes for selection state
- Include clear focus indicators for keyboard navigation
- Consider screen reader users when designing chip choice groups

**Main Component Classes:**

- `.MuiChip-icon` - Leading icon container

**State Classes:**

- `.Mui-disabled` - Applied when the disabled prop is true

- `.not-selected` - Applied when state is not selected

**Variant Classes:**

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(1) - Padding between elements
- theme.typography.body2 - Text style for chip labels
- theme.components.MuiChip.styleOverrides.root - Override for chip root element
- theme.components.MuiChip.styleOverrides.label - Override for chip label element
- theme.components.MuiChip.styleOverrides.icon - Override for icon element

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior using breakpoint values.

**Layout Specifications:**

- Border radius: Based on size type (sm, md, lg, xl)
- Label padding: `2px` (default), `0px` (compressed)
- Font size: Based on size type (xs, sm, md, lg, xl)
