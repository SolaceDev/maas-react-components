<Subtitle>Component Description</Subtitle>

The Solace Select component provides a customizable dropdown selection field with support for various display formats and states. It's useful for creating accessible and feature-rich selection interfaces, allowing users to choose from a predefined list of options with support for icons, subtexts, and custom styling.

The component implements a flexible selection pattern with the purpose of handling both simple and complex option displays. This pattern helps users make selections efficiently while maintaining accessibility and providing rich visual feedback through various states and customization options.

<Subtitle>When To Use It</Subtitle>

Use Solace Select when you need to provide users with a way to choose from a predefined list of options. This selection pattern is particularly effective for forms and configuration interfaces.

Common use cases include:

- Selecting from a list of predefined options
- Choosing configuration settings
- Filtering content by category
- Setting preferences or parameters
- Creating form controls with validation
- Implementing environment or status selectors

<Subtitle>When Not To Use It</Subtitle>

- When users need to input free-form text (use a text input instead)
- For boolean choices (use a checkbox or toggle)
- When there are fewer than 3 options (consider radio buttons)
- When users need to select multiple items simultaneously (use checkboxes or multi-select)
- When the list of options is extremely long (consider using a searchable dropdown)
- When the selection requires a more complex interface (use a dedicated selection component)

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiSelect-root` - Root select container
- `.MuiSelect-select` - Select input element
- `.MuiSelect-icon` - Dropdown icon container
- `.MuiMenuItem-root` - Menu item container

**State Classes:**

- `.Mui-focused` - Applied when the select has focus
- `.Mui-disabled` - Applied when the select is disabled
- `.Mui-error` - Applied when hasErrors is true
- `.readOnlySelect` - Applied when readOnly is true
- `.emptySelect` - Applied when no value is selected

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.spacing(4)` - Height of the select input
- `theme.palette.text.primary` - Text color
- `theme.palette.error.main` - Error state color
- `theme.palette.action.disabled` - Disabled state color

**Layout Specifications:**

- Select height: `theme.spacing(4)`
- Menu max height: Customizable through maxHeight prop
- Menu width: Matches select width by default
- Menu item padding: Inherits from MUI MenuItem
- Transition: Uses Collapse transition for menu animation
