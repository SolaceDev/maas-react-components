<Subtitle>Component Description</Subtitle>

The Solace Empty State Banner component provides a visually rich, engaging layout for empty states with branded gradients, imagery, and call-to-action buttons. It's designed to guide users through onboarding flows, feature introductions, or when no content is available, offering clear pathways to get started.

The component implements a full-width banner pattern with integrated imagery, typography hierarchy, and dual action buttons. This pattern helps transform potentially frustrating empty states into opportunities for user engagement and feature discovery.

<Subtitle>When To Use It</Subtitle>

Use Solace Empty State Banner when you need to create an engaging, branded empty state experience that encourages user action. This component is particularly effective for:

- Welcome screens and onboarding experiences
- Feature introduction or marketing moments within the application
- Main dashboard empty states when users haven't created any content yet
- Product tour starting points or feature discovery screens
- Landing pages for new user experiences
- Empty state pages that need to feel welcoming rather than empty
- Promotional banners for new features or capabilities
- Getting started flows that require user education

<Subtitle>When Not To Use It</Subtitle>

- For inline empty states within data tables or lists - use SolaceDetailMessage instead
- When space is limited or the context is secondary - use simpler empty state components
- For error states or system messages - use SolaceMessageBox or SolaceErrorBox
- In dense interfaces where the banner would be overwhelming - use compact alternatives
- For temporary loading states - use progress indicators instead
- When you need simple, quick empty states without branding - use SolaceDetailMessage
- In forms or narrow content areas where the banner would dominate the interface
