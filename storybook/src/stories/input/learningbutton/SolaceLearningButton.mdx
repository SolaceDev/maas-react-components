<Subtitle>Component Description</Subtitle>

The Solace Learning Button component provides a customizable button for interactive learning experiences. It's useful for creating engaging call-to-action elements, allowing users to initiate learning processes or navigate through educational content.

The component implements a flexible button system with the purpose of enhancing user interaction in learning environments. This pattern helps users clearly identify actionable items while maintaining a consistent and appealing visual style across the application.

<Subtitle>When To Use It</Subtitle>

Use Solace Learning Button when you need to create interactive elements in learning-focused interfaces. This button pattern is particularly effective for guiding users through educational content or initiating learning processes.

Common use cases include:

- Starting a learning module or course
- Navigating through steps in a tutorial
- Submitting answers or completing learning tasks
- Accessing additional learning resources
- Initiating interactive learning experiences
- Highlighting key actions in educational interfaces

<Subtitle>When Not To Use It</Subtitle>

- For general navigation that doesn't relate to learning experiences (use standard navigation components)
- When a more subtle interaction is needed (consider using a text link instead)
- For form submissions that aren't part of a learning process (use a standard submit button)
- When the action doesn't align with the learning-focused theme of the component
- In dense interfaces where a full button might be too prominent (consider icon buttons or text links)
- For destructive actions or warnings (use a dedicated warning or danger button component)

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiButton-root` - Root button container
- `.MuiIconButton-root` - Root icon button container (for icon variant)
- `.call-to-action-button` - Applied to call-to-action variant
- `.dark-call-to-action-button` - Applied to dark call-to-action variant
- `.dark-outline-button` - Applied to dark outline variant

**State Classes:**

- `.Mui-disabled` - Applied when the button is disabled
- `.Mui-focusVisible` - Applied when the button has keyboard focus

**Variant Classes:**

- `.MuiButton-contained` - Applied to call-to-action and dark-call-to-action variants
- `.MuiButton-outlined` - Applied to dark-outline variant
- `.icon-button` - Applied to icon variant

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.primary.main` - Main color for call-to-action buttons
- `theme.palette.ux.accent.n2.wMain` - Used for focus outlines

**Layout Specifications:**

- Button padding: Varies based on variant and presence of icons
- Icon button size: `large`
- Transition: Inherits MUI's default transition for smooth state changes
