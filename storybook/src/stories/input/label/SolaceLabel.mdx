<Subtitle>Component Description</Subtitle>

The Solace Label component provides a customizable label for form elements. It's useful for creating accessible and visually consistent form labels, allowing users to clearly identify and understand the purpose of associated input fields.

The component implements a flexible labeling system with the purpose of enhancing form usability and accessibility. This pattern helps users navigate forms more efficiently while still maintaining a clean and organized layout.

<Subtitle>When To Use It</Subtitle>

Use Solace Label when you need to provide a descriptive text label for form inputs. This labeling pattern is particularly effective for creating clear and accessible user interfaces.

Common use cases include:

- Labeling text input fields in forms
- Providing descriptions for checkboxes or radio buttons
- Creating section headers in complex forms
- Indicating required fields with visual cues

<Subtitle>When Not To Use It</Subtitle>

- When you need to display large blocks of explanatory text (use a paragraph or description component instead)
- For clickable text that performs an action (use a button or link component)
- As a replacement for page titles or headings (use appropriate heading components)
- When the label text is very long and might disrupt the form layout (consider using a tooltip or help text)
- For purely decorative text that doesn't describe an input (use a regular text component)
- When you want to hide labels visually but keep them accessible (use a specialized visually-hidden component)

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiFormLabel-root` - Root label container
- `.MuiFormLabel-colorPrimary` - Primary color styling
- `.MuiFormLabel-filled` - Applied when the associated input is filled

**State Classes:**

- `.Mui-disabled` - Applied when the label is disabled
- `.Mui-error` - Applied when there's an error state
- `.Mui-focused` - Applied when the associated input is focused
- `.Mui-required` - Applied when the label is for a required field
- `.read-only` - Applied when the readOnly prop is true

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.text.primary` - Main label text color
- `theme.palette.text.disabled` - Color for disabled state

**Layout Specifications:**

- Label display: `block`
- White space: `normal` (or `nowrap` if noWrap prop is true)
- Transition: Inherits MUI's default transition for smooth state changes
