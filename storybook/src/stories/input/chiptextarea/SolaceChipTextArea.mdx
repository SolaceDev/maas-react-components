<Subtitle>Component Description</Subtitle>

The Solace Chip Text Area component provides a multi-line text input interface that automatically converts user input into visual chips based on delimiter characters (comma, semicolon, or space). It's useful for applications where users need to enter multiple values like email addresses, tags, or keywords, allowing users to manage collections of text values in an intuitive chip-based interface.

The component implements a tag input pattern with the purpose of making multi-value entry efficient and visually organized. This pattern helps users add, edit, and remove multiple text values through automatic tokenization while maintaining form consistency and providing accessible multi-value input workflows.

<Subtitle>When To Use It</Subtitle>

Use Solace Chip Text Area when you need users to input multiple discrete text values with visual feedback. This input pattern is particularly effective for applications requiring collection management and multi-value data entry.

Common use cases include:

- Email recipient lists where users enter multiple email addresses separated by delimiters for mailing functionality
- Tag and keyword entry for content categorization, search optimization, or metadata management
- Multi-value form fields like skills, interests, or categories where users add multiple related items
- Contact list management for adding multiple phone numbers, addresses, or other contact information
- Configuration settings where users specify multiple values like server names, domains, or file paths
- Search filter inputs where users can add multiple search terms or criteria as individual chips
- Permission and access control where users specify multiple users, roles, or resource identifiers

<Subtitle>When Not To Use It</Subtitle>

- For single-value text input without tokenization - use SolaceTextField for simple text entry instead
- When users need to enter long-form text content - use SolaceTextArea for paragraph or document text
- For predefined option selection - use SolaceSelect or SolaceAutocomplete with multiple selection instead
- When chip deletion should be restricted or controlled - consider using read-only chips or controlled deletion patterns
- For numeric value ranges or calculations - use specialized numeric input components instead
- When validation needs are complex or require server-side verification - implement custom validation workflows
- For mobile-first applications where chip manipulation is difficult - consider simplified list interfaces or alternative input methods
- When accessibility requires keyboard-only navigation of individual chips - ensure proper ARIA support and focus management

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiInputBase-root` - Root input container for the chip text area
- `.MuiInputBase-input` - Input field element within the chip text area
- `.MuiChip-root` - Individual chip elements created from user input
- `.MuiChip-deleteIcon` - Delete icon within each removable chip

**State Classes:**

- `.error` - Applied to container when chips have validation errors or hasErrors is true
- `.Mui-focused` - Applied when the input area has keyboard focus
- `.Mui-disabled` - Applied when the component is in disabled state
- `.solaceChip` - Applied to all chip elements for consistent Solace styling

**Custom Classes:**

- `[data-chip="true"]` - Applied to chip container elements for event handling and styling
- `.error-status` - Applied to chips that fail validation checks
- `.no-status` - Applied to chips that pass validation successfully

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(4) - Container min-height for consistent form field height
- theme.typography.body1 - Text style for input and chip text
- theme.components.MuiInputBase.styleOverrides.root - Base input styling
- theme.components.MuiChip.styleOverrides.root - Chip styling

Available mixins for consistent styling:

- theme.mixins.chip.base - Standard chip styling patterns

**Layout Specifications:**

- Container min-height: `theme.spacing(4)` (32px)
- Container padding: `2px 4px`
- Container border: `1px solid` with dynamic color
- Container border-radius: `4px`
- Chip margin: `theme.spacing(0.5)` (4px)
- Input wrapper min-width: `50px`
- Input field height: `120px` (4 rows maximum)
- Chip layout: `inline-flex` with `flex-start` alignment
- Transition duration: `200ms` for hover and focus effects
