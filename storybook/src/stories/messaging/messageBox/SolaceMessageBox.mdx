<Subtitle>Component Description</Subtitle>

The Solace Message Box component provides a dismissible notification banner for displaying important messages to users with different severity levels. It's useful for communicating system status, validation errors, warnings, or success confirmations in a prominent but non-intrusive way, allowing users to acknowledge and dismiss messages when appropriate.

The component implements a message notification pattern with the purpose of providing contextual feedback about system state or user actions. This pattern helps users understand the outcome of their actions or be aware of important system information while maintaining workflow continuity.

<Subtitle>When To Use It</Subtitle>

Use Solace Message Box when you need to display important information that requires user awareness but doesn't block their workflow. This message pattern is particularly effective for persistent notifications that remain visible until dismissed.

Common use cases include:

- Displaying form validation errors or field-level feedback
- Showing system status messages or configuration warnings
- Confirming successful operations like saves or updates
- Alerting users to important information or policy changes
- Providing contextual help or informational guidance
- Displaying error states that don't require immediate action

<Subtitle>When Not To Use It</Subtitle>

- For critical errors that require immediate action (use modal dialogs instead)
- For temporary notifications that should auto-dismiss (use toast notifications instead)
- When the message blocks user workflow (use inline validation instead)
- For promotional or marketing content (use learning banners instead)
- When messages need to persist across page navigation (use global notifications instead)
- For progress updates or loading states (use progress indicators instead)

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `InfoBoxContainer` - Root message box container with variant styling
- `InfoBoxMessageContainer` - Container for message content and close button
- `InfoBoxMessage` - Main message content area with icon and text
- `MessageTextContainer` - Text content with color styling
- `IconContainer` - Container for variant-specific icons
- `DetailsContainer` - Container for additional details content

**State Classes:**

- Message box conditionally renders based on internal `open` state
- `.dense` class applied when dense prop is true for compact spacing

**Variant Classes:**

- `.info` - Applied for informational messages
- `.error` - Applied for error messages
- `.warn` - Applied for warning messages
- `.success` - Applied for success messages

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.mixins.component_MessageBox.container` - Main container styling
- `theme.mixins.component_MessageBox.messageContainer` - Message container layout
- `theme.mixins.component_MessageBox.messageTextContainer` - Text styling and color
- `theme.mixins.component_MessageBox.iconContainer` - Icon positioning and spacing
- `theme.palette.ux.info.w100` - Info variant color
- `theme.palette.ux.error.w100` - Error variant color
- `theme.palette.ux.warning.w100` - Warning variant color
- `theme.palette.ux.success.w100` - Success variant color

**Layout Specifications:**

- Icon size: 24px for all variant icons
- Close button: 24px icon with icon variant styling
- Dense mode: Reduced vertical padding for compact display
- Details section: Separate container below main message
- Responsive layout with proper spacing between elements
