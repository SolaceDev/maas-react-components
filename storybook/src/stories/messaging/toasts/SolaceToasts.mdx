<Subtitle>Component Description</Subtitle>

The Solace Toasts component provides temporary notification messages that appear at the bottom center of the screen to communicate the outcome of user actions or system events. It's useful for providing immediate feedback about operations without interrupting the user's workflow, allowing users to continue their tasks while being informed of important updates.

The component implements a toast notification pattern with the purpose of delivering timely, non-intrusive feedback. This pattern helps users understand the results of their actions through brief, automatically dismissing messages that don't block the interface or require explicit dismissal.

<Subtitle>When To Use It</Subtitle>

Use Solace Toasts when you need to provide temporary feedback about user actions or system events. This notification pattern is particularly effective for confirming actions and providing status updates.

Common use cases include:

- Confirming successful operations like saves, updates, or deletions
- Notifying users of background process completion
- Displaying brief error messages that don't require immediate action
- Showing undo actions for reversible operations
- Communicating system status changes or connectivity issues
- Providing feedback for bulk operations or batch processes

<Subtitle>When Not To Use It</Subtitle>

- For critical errors that require immediate user attention (use modal dialogs instead)
- When messages need to persist until user acknowledgment (use message boxes instead)
- For complex information that requires reading time (use alerts or dedicated sections instead)
- When users need to reference the message later (use persistent notifications instead)
- For promotional or educational content (use banners instead)
- When the message relates to specific form fields (use inline validation instead)

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiSnackbar-root` - Root snackbar container
- `.MuiSnackbar-anchorOriginBottomCenter` - Positioning for bottom center display
- `.alert-toast` - Custom className for toast styling
- `.MuiAlert-root` - Alert component when severity is provided
- `.MuiAlert-icon` - Container for severity icons

**Severity Classes:**

- `.MuiAlert-standardSuccess` - Applied for success toast styling
- `.MuiAlert-standardError` - Applied for error toast styling
- `.MuiAlert-standardInfo` - Applied for info toast styling
- `.MuiAlert-standardWarning` - Applied for warning toast styling

**Custom Classes:**

- Custom padding applied based on severity presence
- Icon mapping with theme-based colors

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.brand.wMain` - Success icon color
- `theme.palette.ux.error.w100` - Error icon color
- `theme.palette.ux.info.w100` - Info icon color
- `theme.palette.ux.warning.w100` - Warning icon color

**Layout Specifications:**

- Anchor position: Bottom center of viewport
- Auto-hide duration: 4000ms (4 seconds) without action, 8000ms (8 seconds) with action
- Resume hide duration: 1000ms after user interaction
- Padding: `0 16px 0 8px` with severity, `0 16px` without severity
- Click-away dismissal: Disabled for better user control
- Role: `toast` for accessibility
