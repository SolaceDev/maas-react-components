<Subtitle>Component Description</Subtitle>

The Solace Typography component provides consistent text styling and formatting across the Solace design system. It's useful for displaying text content with standardized typography variants, allowing users to maintain visual hierarchy and design consistency.

The component implements Material-UI's Typography component with custom Solace theming and color mappings. This pattern ensures consistent text rendering while providing semantic color options for different states and contexts.

<Subtitle>When To Use It</Subtitle>

Use Solace Typography when you need to display text content with consistent styling and semantic meaning. This component is particularly effective for maintaining design system compliance and ensuring proper text hierarchy.

Common use cases include:

- Page headings and subheadings (h1-h5 variants)
- Body text and paragraph content (body1 variant)
- Caption text and fine print (caption variant)
- Status messages with semantic colors (success, error, warning, info)
- Card titles and content text
- Form labels and helper text
- List items and navigation text

<Subtitle>When Not To Use It</Subtitle>

- Don't use for interactive text elements - use SolaceButton with text variant instead
- Avoid for input field text - use appropriate form components like SolaceTextField
- Don't use for highly styled decorative text - consider custom styled components
- Avoid for text that needs complex formatting - use rich text editors or HTML elements
- Don't use for code snippets - use proper code formatting components
- Avoid for text that requires custom fonts outside the design system - use standard HTML elements with custom styling
- Don't use for text inside tooltips - SolaceTooltip handles its own text styling

**Theme Customization:**

**Color Theming:**
The component uses the theme.palette.ux color system for all color-related styling. Key color theme paths:

- theme.palette.ux.info.w100 - Info state color
- theme.palette.ux.error.w100 - Error state color
- theme.palette.ux.warning.w100 - Warning state color
- theme.palette.ux.success.w100 - Success state color
- theme.palette.ux.primary.text.wMain - Primary text color
- theme.palette.ux.secondary.text.wMain - Secondary text color

To customize colors, override the palette.ux values in your theme configuration by extending the base theme and modifying the specific color group and variant you want to change.

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.typography.fontFamily - Primary font family
- theme.typography.h1 - Heading 1 typography definitions (size, weight, line height)
- theme.typography.h2 - Heading 2 typography definitions
- theme.typography.h3 - Heading 3 typography definitions
- theme.typography.h4 - Heading 4 typography definitions
- theme.typography.h5 - Heading 5 typography definitions
- theme.typography.body1 - Body text typography definitions
- theme.typography.caption - Caption text typography definitions
- theme.components.MuiTypography.styleOverrides.root - Override typography styling

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify typography scales, weights, and spacing using the theme typography system, and customize appearance through typography overrides.

**Layout Specifications:**

- Default variant: body1
- Default color: textPrimary
- Supports all Material-UI Typography spacing props
- Inherits theme typography scales and weights
- Responsive typography based on theme breakpoints
