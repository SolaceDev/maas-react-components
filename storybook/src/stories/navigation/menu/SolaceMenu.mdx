<Subtitle>Component Description</Subtitle>

The Solace Menu component provides a dropdown menu interface for displaying a list of actions or options. It combines a trigger button with a contextual menu that appears on click, allowing users to access multiple related actions in a space-efficient manner.

The component implements a menu pattern with the purpose of organizing multiple actions or options under a single trigger. This pattern helps reduce interface clutter while providing easy access to contextual actions and commands.

<Subtitle>When To Use It</Subtitle>

Use Solace Menu when you need to group multiple actions or options under a single trigger. This menu pattern is particularly effective for organizing related functionality.

Common use cases include:

- Action menus with multiple related operations (edit, delete, duplicate, etc.)
- Context menus for table rows or card items
- Settings and configuration options
- More actions (...) menus when primary actions are limited
- Nested menu structures for hierarchical options
- Filter and sort options in lists or tables

<Subtitle>When Not To Use It</Subtitle>

- For primary navigation between pages - use navigation components or tabs instead
- When there's only one action - use a direct button instead
- For form inputs and selections - use select components or radio buttons
- When all options should be immediately visible - use button groups or tabs
- For binary choices - use toggle buttons or switches
- When actions are destructive and need confirmation - consider modal dialogs

- `.Mui-selected` - Applied to selected menu items
- `.Mui-disabled` - Applied to disabled menu items
- `.Mui-focusVisible` - Applied when menu item has keyboard focus
- `.pressed` - Applied to button when menu is open

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing(1, 0) - Menu container padding
- theme.spacing(1) - Gap between icon and text
- theme.components.MuiMenu.styleOverrides.paper - Override menu container styling
- theme.components.MuiMenuItem.styleOverrides.root - Override menu item styling
- theme.typography.body2 - Menu item text styling

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and positioning through menu overrides.

**Layout Specifications:**

- Default max width: 280px (configurable via maxWidth prop)
- Default max height: Based on numOfMenuItemDisplayed (default 9 items)
- Item height: 38px for single-line, 58px for multiline
- Menu positioning: Configurable via anchorOrigin and transformOrigin
- Padding: theme.spacing(1, 0) for menu container
- Icon spacing: theme.spacing(1) gap between icon and text
