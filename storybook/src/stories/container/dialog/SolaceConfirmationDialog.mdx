<Subtitle>Component Description</Subtitle>

The Solace Confirmation Dialog component provides modal dialogs that appear in front of the main content to provide critical information, require decisions, or involve multiple tasks. It's useful for keeping users in context when quick actions are needed, allowing users to make decisions without losing their current workflow.

The component implements a flexible dialog pattern with the purpose of drawing user attention to specific actions or information. This pattern helps users stay focused on important decisions while still maintaining their context within the application.

<Subtitle>When To Use It</Subtitle>

Use Solace Confirmation Dialog when you have critical information to display or important decisions that require user acknowledgment. This interaction pattern is particularly effective for workflow-critical confirmations and contextual information display.

Common use cases include:

- Confirming destructive actions like deleting items or clearing data
- Collecting user input through form fields without navigating away
- Displaying critical alerts or warnings that require acknowledgment
- Presenting quick tasks or decisions that progress the workflow

<Subtitle>When Not To Use It</Subtitle>

- Complex multi-step interactions that require extensive navigation
- When context from the rest of the screen is needed for decision making
- Don't open dialogs without user interaction or clear triggering events
- Avoid for non-critical information that could be displayed inline
- Don't use for simple notifications that don't require user decisions
- Avoid when the interaction disrupts the user flow unnecessarily

**Non-Color Theming:**
Structural and layout customization through component overrides and mixins:

- theme.spacing([NUMBER]) - Padding and margin (e.g., theme.spacing(3, 3, 2, 3) for title padding)
- theme.typography.[VARIANT] - Text styles for dialog content and title
- theme.breakpoints.values.[SIZE] - Responsive behavior breakpoints
- theme.components.MuiDialog.styleOverrides.root - Custom styling for dialog container
- theme.components.MuiDialogTitle.styleOverrides.root - Custom styling for dialog title
- theme.components.MuiDialogContent.styleOverrides.root - Custom styling for dialog content
- theme.components.MuiDialogActions.styleOverrides.root - Custom styling for dialog actions

Available mixins for consistent styling:

- theme.mixins.toolbar - Consistent toolbar styling if applicable in dialog actions

To customize layout and structure, override the component styleOverrides in your theme configuration. You can modify spacing using the theme spacing function, typography using predefined variants, and responsive behavior using breakpoint values.

**Layout Specifications:**

- Dialog title padding: `theme.spacing(3, 3, 2, 3)` (24px 24px 16px 24px)
- Dialog content: Flexible display based on contentLayout prop
- Action buttons: Right-aligned with consistent spacing
- Custom actions: Left-aligned with auto margin
- Transition timeout: `225ms`
