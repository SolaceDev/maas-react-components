<Subtitle>Component Description</Subtitle>

The Solace Toggle Button Group component provides a set of mutually exclusive toggle buttons for selection. It's useful for filtering, categorization, or switching between different view states, allowing users to select one option from a group of related choices.

The component implements an exclusive selection pattern with the purpose of providing clear visual feedback for the active state. This pattern helps users understand their current selection while still providing easy access to switch between options.

<Subtitle>When To Use It</Subtitle>

Use Solace Toggle Button Group when you have a small set of mutually exclusive options that users need to switch between. This selection pattern is particularly effective for filtering content or switching between different view modes.

Common use cases include:

- Filtering data by category (All, Published, Draft)
- Switching between view modes (List, Grid, Card)
- Toggling between data timeframes (Today, Week, Month)
- Selecting content types for display (Articles, Videos, Images)
- Choosing subscription states (All, Active, Inactive)

<Subtitle>When Not To Use It</Subtitle>

- When you have more than 5-6 options - use a dropdown select instead
- For boolean true/false states - use a single toggle switch or checkbox
- When multiple selections are allowed - use checkboxes or multi-select components
- For navigation between pages - use tabs or navigation links instead
- When options are too lengthy to fit in button format - use radio buttons
- For actions that trigger immediate effects - use regular action buttons

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- `.MuiToggleButtonGroup-root` - Root toggle button group container
- `.MuiToggleButtonGroup-grouped` - Individual toggle button elements
- `.MuiToggleButton-root` - Individual button styling

**State Classes:**

- `.Mui-selected` - Applied when a toggle button is selected/active
- `.Mui-disabled` - Applied when the button group or individual button is disabled
- `.Mui-focusVisible` - Applied when a button receives keyboard focus

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.brand.w10` - Selected button background color
- `theme.palette.ux.primary.wMain` - Selected button text color
- `theme.palette.ux.secondary.w40` - Button border color
- `theme.palette.ux.background.w10` - Default button background
- `theme.palette.ux.secondary.w10` - Hover state background

**Layout Specifications:**

- Button height: `32px` (theme.spacing(4))
- Button padding: `0 16px` (theme.spacing(0, 2))
- Border style: `1px solid` with theme colors
- Font weight: Regular for unselected, Medium for selected
- Text transform: `none` (preserves original casing)
