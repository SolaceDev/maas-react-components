<Subtitle>Component Description</Subtitle>

The Solace Icon Tabs component provides icon-based navigation tabs for switching between different views or modes. It's useful for displaying view toggles with visual icons, allowing users to switch between different presentation modes or content views efficiently.

The component implements an icon-based tab navigation pattern with the purpose of providing space-efficient view switching. This pattern helps users quickly identify and switch between different display modes while conserving horizontal space.

<Subtitle>When To Use It</Subtitle>

Use Solace Icon Tabs when you need compact, icon-based view switching. This navigation pattern is particularly effective for toggling between different content presentations.

Common use cases include:

- View mode toggles (list view vs. grid view)
- Content presentation switches (graph view vs. table view)
- Filter or display mode selection
- Compact navigation in constrained spaces like toolbars

<Subtitle>When Not To Use It</Subtitle>

- For primary navigation with multiple options - use standard tabs with text labels instead
- When icons might be ambiguous to users - include text labels for clarity
- For navigation between different pages or sections - use breadcrumbs or standard navigation
- When there are more than 3-4 options - consider a dropdown menu or standard tabs
- For one-time actions rather than view modes - use buttons instead
- When accessibility requires clear text labels - combine with text or use standard tabs

<Subtitle>CSS Classes / Styles</Subtitle>

**Main Component Classes:**

- Custom `ViewsContainer` - Flex container for icon tabs
- Custom `ViewButton` - Individual icon tab button
- Custom `ViewActive` - Active state indicator bar
- Custom `ViewIcon` - Icon container with padding

**State Classes:**

- `active` prop styling - Background, border, and color changes for active state
- `:hover` styling - Background color and text color changes on hover

**Custom Classes:**

- Active tab: `background-color: background.w10`, `border-inline: 1px solid secondary.w20`
- Inactive tab: `background-color: background.w20`, `border-bottom: 1px solid secondary.w40`
- Hover state: `background-color: secondary.w10`, `color: primary.text.wMain`

**Theme Customization:**
The component uses Material-UI's theme system with custom Solace styling. Key theme paths:

- `theme.palette.ux.background.w10` - Active tab background
- `theme.palette.ux.background.w20` - Inactive tab background
- `theme.palette.ux.secondary.w20` - Active tab border
- `theme.palette.ux.accent.n2.wMain` - Active indicator color

**Layout Specifications:**

- Container height: `theme.spacing(5)` (40px)
- Icon padding: `theme.spacing(1, 1)` (8px)
- Active indicator: `height: 2px`, `width: 100%`
- Border styling: `1px solid` for active state
- Cursor: `pointer` for inactive, `default` for active
